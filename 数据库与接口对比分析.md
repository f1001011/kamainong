# Kamainong 数据库与 Honeywell 前端接口对比分析

## 一、数据格式差异分析

### 1. 用户表 (ntp_common_user)

**前端需要的字段：**
```typescript
interface User {
  id: number;
  phone: string;
  nickname: string | null;
  avatar: string | null;
  availableBalance: string;  // Decimal 格式
  frozenBalance: string;     // Decimal 格式
  totalEarnings?: string;
  totalRecharge?: string;
  totalWithdraw?: string;
  inviteCode: string;
  vipLevel: number;
  svipLevel: number;
  firstPurchaseDone: boolean;
  status: 'ACTIVE' | 'BANNED';
  createdAt: string;
}
```

**数据库现有字段对比：**
- ✅ `id` - 存在
- ✅ `phone` - 存在 (user_phone)
- ❌ `nickname` - **缺失**
- ❌ `avatar` - **缺失**
- ✅ `availableBalance` - 存在 (money)
- ❌ `frozenBalance` - **缺失**
- ✅ `inviteCode` - 存在 (invitation_code)
- ❌ `vipLevel` - **缺失** (需要从 ntp_common_vip_log 关联)
- ❌ `svipLevel` - **缺失**
- ❌ `firstPurchaseDone` - **缺失**
- ✅ `status` - 存在 (status: 1正常/2封禁)
- ✅ `createdAt` - 存在 (create_time)

**需要补充的字段：**
```sql
ALTER TABLE `ntp_common_user` 
ADD COLUMN `nickname` varchar(50) DEFAULT NULL COMMENT '昵称',
ADD COLUMN `avatar` varchar(255) DEFAULT NULL COMMENT '头像URL',
ADD COLUMN `frozen_balance` decimal(12,2) DEFAULT 0.00 COMMENT '冻结余额',
ADD COLUMN `first_purchase_done` tinyint(1) DEFAULT 0 COMMENT '是否完成首购';
```

## 三、数据格式转换规则

### 1. 金额字段
- **数据库**: `decimal(12,2)` 
- **前端**: `string` (如 "100.00")
- **转换**: PHP 使用 `number_format($amount, 2, '.', '')`

### 2. 时间字段
- **数据库**: `datetime` (如 2026-03-24 12:00:00)
- **前端**: ISO 8601 字符串 (如 2026-03-24T12:00:00+01:00)
- **转换**: 需要根据系统时区转换

### 3. 状态字段映射

#### 用户状态
- 数据库: `1`正常 / `2`封禁
- 前端: `ACTIVE` / `BANNED`

#### 产品状态
- 数据库: `1`上架 / `2`即将推出 / `0`下架
- 前端: `AVAILABLE` / `COMING_SOON` / `SOLD_OUT`

#### 订单状态
- 充值: `0`待支付 / `1`成功 / `2`失败 / `3`取消
- 前端: `PENDING` / `SUCCESS` / `FAILED` / `CANCELLED`

#### 任务状态
- 数据库: `is_completed=0, is_claimed=0` → 前端: `LOCKED`
- 数据库: `is_completed=1, is_claimed=0` → 前端: `CLAIMABLE`
- 数据库: `is_completed=1, is_claimed=1` → 前端: `CLAIMED`
