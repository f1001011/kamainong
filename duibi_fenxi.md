# 数据库与文档对比分析

## 一、数据库表结构分析

### 1. 商品表 (ntp_common_goods)

**现有字段**：
- `goods_name` - 商品名称
- `goods_money` - 投注价格
- `day_red` - 每日分红
- `period` - 投资周期
- `status` - 状态（1上架，2即将推出，0下架）
- `is_examine` - 是否新手体验产品
- `buy_num` - 可购买次数（0无限次）
- `level_vip` - 可购买等级限制
- `goods_agent_1/2/3` - 一二三级返佣

**✅ 满足需求**：
- 支持产品名称、价格、周期
- 支持每日分红配置
- 支持状态管理（可购买/即将推出）
- 支持购买次数限制
- 支持代理返佣

**❌ 缺失功能**：
- 缺少"收益次数/天"字段（文档中每天3次、5次等）
- 缺少"单次收益"字段
- 缺少"年化收益率"字段


### 2. 用户表 (ntp_common_user)

**现有字段**：
- `level_vip` - VIP等级
- `money_balance` - 用户余额
- `money_team` - 团队佣金
- `agent_id_1/2/3` - 一二三级代理
- `total_recharge` - 累计充值
- `total_red` - 分红

**✅ 满足需求**：
- 支持VIP等级
- 支持代理层级关系
- 支持充值和分红记录

**✅ 满足需求**：
- 已有VIP等级字段（对应文档中的SVIP1-10）

**❌ 缺失功能**：
- 缺少每日VIP奖励记录


### 3. VIP表 (ntp_common_vip)

**现有字段**：
- `vip` - VIP等级（即文档中的SVIP等级）
- `experience` - 需要经验

**✅ 满足需求**：
- 支持VIP等级（对应SVIP1-10）

**❌ 缺失功能**：
- 缺少每日奖励金额配置
- 缺少升级条件（购买2个同等级产品）

## 二、功能对比总结

### ✅ 已满足的功能

1. **产品体系**
   - 产品名称、价格、周期配置 ✓
   - 状态管理（可购买/即将推出）✓
   - 购买次数限制 ✓
   - 新手体验产品标识 ✓

2. **代理体系**
   - 三级代理关系 ✓
   - 代理返佣配置 ✓
   - 团队佣金记录 ✓

3. **用户系统**
   - 用户余额管理 ✓
   - 充值提现记录 ✓
   - 分红记录 ✓


### ❌ 缺失的核心功能

#### 1. 产品收益机制
- **缺少字段**：每天收益次数（3次、5次、10次等）
- **缺少字段**：单次收益金额
- **缺少字段**：年化收益率
- **影响**：无法实现文档中"每天多次领取"的机制

#### 2. VIP会员体系（文档中的SVIP）
- **已有表**：ntp_common_vip（VIP配置表）✓
- **已有字段**：用户表中的level_vip ✓
- **缺少字段**：VIP每日奖励金额配置
- **缺少逻辑**：购买2个同等级产品升级VIP
- **缺少表**：VIP每日奖励领取记录表
- **影响**：VIP等级存在，但缺少每日奖励机制


#### 3. LV1月薪奖励
- **缺少表**：月薪奖励配置表
- **缺少表**：月薪领取记录表
- **缺少逻辑**：团队LV1累计充值统计
- **缺少逻辑**：每月只能领取最高档位
- **影响**：无法实现月薪奖励功能

#### 4. 代理级别体系
- **缺少表**：代理级别配置表（见习组长、市场主管等7个级别）
- **缺少字段**：用户当前代理级别
- **缺少逻辑**：根据充值会员数量自动晋升
- **影响**：无法实现7级代理体系


#### 5. 每日奖池活动
- **缺少表**：奖池配置表
- **缺少表**：每日获奖记录表
- **缺少逻辑**：每天评选一二三等奖
- **缺少逻辑**：奖池金额管理（每天3000 XAF）
- **影响**：无法实现每日奖池活动

#### 6. 收益领取规则
- **缺少表**：收益领取记录表（记录每次领取）
- **缺少逻辑**：24小时保留期机制
- **缺少逻辑**：超时作废机制
- **影响**：无法实现手动领取和过期作废

## 三、数据库改进建议

### 需要新增的表

1. **VIP配置表** (ntp_common_vip) 新增字段
   - daily_reward (每日奖励金额)
   - required_product_id (需要购买的产品ID)
   - required_count (需要购买数量，固定为2)

2. **VIP每日奖励领取记录表** (ntp_common_vip_daily_reward_log)
   - user_id (用户ID)
   - vip_level (VIP等级)
   - reward_amount (奖励金额)
   - claim_date (领取日期)
   - create_time (创建时间)

3. **月薪奖励配置表** (ntp_common_monthly_salary_config)
   - team_recharge_amount (团队LV1累计充值要求)
   - reward_amount (奖励金额)

4. **代理级别配置表** (ntp_common_agent_level_config)
   - level_name (级别名称：见习组长、市场主管等)
   - required_members (需要的充值会员数量)
   - member_type (会员类型：LV1或LV123)
   - reward_amount (一次性奖励金额)


5. **奖池配置表** (ntp_common_prize_pool_config)
   - daily_amount (每日投入金额 3000)
   - prize_1_amount (一等奖金额 1388)
   - prize_2_amount (二等奖金额 888)
   - prize_3_amount (三等奖金额 688)

6. **收益领取记录表** (ntp_common_income_claim_log)
   - user_id (用户ID)
   - order_id (订单ID)
   - claim_amount (领取金额)
   - claim_time (领取时间)
   - expire_time (过期时间)
   - status (状态：待领取/已领取/已过期)


### 需要修改的表

1. **商品表** (ntp_common_goods) 新增字段
   - `income_times_per_day` int - 每天收益次数
   - `income_per_time` decimal - 单次收益金额
   - `annual_rate` decimal - 年化收益率

2. **用户表** (ntp_common_user) 新增字段
   - `agent_level` int - 代理级别 (0-7)
   - `agent_level_name` varchar - 代理级别名称

## 四、总体评估

### 完成度评估

| 功能模块 | 完成度 | 说明 |
|---------|--------|------|
| 基础产品体系 | 70% | 缺少收益次数、单次收益字段 |
| 收益领取规则 | 30% | 缺少领取记录和过期机制 |
| VIP会员体系 | 50% | 有VIP等级，缺少每日奖励配置和记录 |
| LV1月薪奖励 | 0% | 完全缺失 |
| 代理级别体系 | 40% | 有三级代理，缺7级代理配置 |
| 每日奖池活动 | 0% | 完全缺失 |

### 总体结论

**当前数据库结构约满足文档需求的 40%**

**主要问题**：
1. VIP体系存在但缺少每日奖励配置和领取记录
2. 缺少月薪奖励和奖池活动的配置表
3. 产品表缺少收益次数相关字段
4. 缺少收益领取的详细记录机制

**建议**：
需要新增6个配置表，修改2个现有表（商品表、VIP表），才能完整实现文档中的所有功能。
