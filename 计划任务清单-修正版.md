# Honeywell Web 计划任务清单（修正版）

## 📋 收益领取机制说明

**重要理解：**
- 用户今天购买产品，明天开始可以领取收益
- 产品有 `income_times_per_day` 字段：每天可领取次数
- 产品有 `minute_claim` 字段：每次领取间隔分钟数
- 如果 `minute_claim = 0`：第二天可以一次性领取全部
- 如果 `minute_claim > 0`：每隔 N 分钟可以领取一次

**示例：**
- Revenu fixe 1: `income_times_per_day=3`, `minute_claim=480`
- 用户 3月17日 23:00 购买
- 3月18日可领取时间：00:00、08:00、16:00（每8小时一次）

## 📋 需要实现的计划任务

### 1. 每分钟任务

#### 1.1 收益可领取状态更新（每分钟）
**功能：** 根据 `minute_claim` 计算并更新可领取的收益

**处理逻辑：**
1. 扫描所有未到期的订单（购买日期 < 今天）
2. 获取产品的 `income_times_per_day` 和 `minute_claim`
3. 计算今天的可领取时间点：
   - 如果 `minute_claim = 0`：00:00 可领取全部
   - 如果 `minute_claim > 0`：计算间隔时间点
4. 检查当前时间是否到达可领取时间
5. 标记对应的收益为"可领取"状态

**执行时间：** 每分钟执行（*/1 * * * *）

**涉及表：**
- `ntp_common_goods_order`
- `ntp_common_goods`

---

### 2. 每日任务

#### 2.1 收益过期处理（每天 00:01）
**功能：** 将超过24小时未领取的收益标记为过期

**处理逻辑：**
- 查找生成时间超过24小时且状态为"可领取"的收益
- 将状态更新为"已过期"

**执行时间：** 每天 00:01:00

**涉及表：**
- `ntp_common_income_claim_log`

---

#### 2.2 奖池开奖（每天 05:00）
**功能：** 计算前一天的奖池获奖者并发放奖励

**处理逻辑：**
- 统计前一天各用户数据：
  - 一等奖：个人LV1团队充值最高（1388 XAF）
  - 二等奖：个人充值最高（888 XAF）
  - 三等奖：个人邀请人数最高（688 XAF）
- 发放奖金到用户余额
- 记录获奖日志

**执行时间：** 每天 05:00:00

**涉及表：**
- `ntp_common_prize_pool_config`
- `ntp_common_prize_pool_log`
- `ntp_common_user`

---

#### 2.3 VIP每日奖励重置（每天 00:00）
**功能：** 重置VIP用户的每日奖励领取状态

**执行时间：** 每天 00:00:00

**涉及表：**
- `ntp_common_vip_daily_reward_log`

---

#### 2.4 转盘次数重置（每天 00:00）
**功能：** 重置用户的每日转盘次数限制

**执行时间：** 每天 00:00:00

**涉及表：**
- `ntp_common_lottery_chance`

---

### 3. 每周任务

#### 3.1 任务进度重置（每周一 00:00）
**功能：** 重置所有用户的任务进度

**处理逻辑：**
- 将所有邀请任务进度清零
- 保留历史完成记录

**执行时间：** 每周一 00:00:00

**涉及表：**
- `ntp_common_task_progress`

---

### 4. 每月任务

#### 4.1 月薪奖励重置（每月1日 00:00）
**功能：** 重置月薪奖励的领取状态

**处理逻辑：**
- 统计上月团队充值数据
- 重置本月领取状态

**执行时间：** 每月1日 00:00:00

**涉及表：**
- `ntp_common_monthly_salary_log`
- `ntp_common_agent_path`

## 📝 实现方式

### Linux Crontab 配置

```bash
# 编辑 crontab
crontab -e

# 每分钟 - 收益可领取状态更新
*/1 * * * * cd /path/to/app_api && php think cron:update_income_status

# 每天00:01 - 收益过期
1 0 * * * cd /path/to/app_api && php think cron:expire_income

# 每天05:00 - 奖池开奖
0 5 * * * cd /path/to/app_api && php think cron:prize_pool

# 每天00:00 - VIP奖励重置
0 0 * * * cd /path/to/app_api && php think cron:reset_vip_reward

# 每天00:00 - 转盘次数重置
0 0 * * * cd /path/to/app_api && php think cron:reset_lottery

# 每周一00:00 - 任务重置
0 0 * * 1 cd /path/to/app_api && php think cron:reset_tasks

# 每月1日00:00 - 月薪重置
0 0 1 * * cd /path/to/app_api && php think cron:reset_salary
```

## 🔧 需要创建的命令

1. `cron:update_income_status` - 收益可领取状态更新（每分钟）
2. `cron:expire_income` - 收益过期处理
3. `cron:prize_pool` - 奖池开奖
4. `cron:reset_vip_reward` - VIP奖励重置
5. `cron:reset_lottery` - 转盘次数重置
6. `cron:reset_tasks` - 任务进度重置
7. `cron:reset_salary` - 月薪重置

---

## ⚠️ 注意事项

1. **时区设置**：确保服务器时区为 UTC+1（喀麦隆时区）
2. **收益计算**：严格按照 `minute_claim` 字段计算可领取时间
3. **24小时规则**：收益生成后24小时内必须领取，否则作废
4. **日志记录**：每个任务执行后记录日志
5. **幂等性**：确保任务可以重复执行而不出错
